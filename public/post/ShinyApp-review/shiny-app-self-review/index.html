Shiny App Self-Review

Last updated on Oct 15, 2020 6 min read R

The Shiny App

The interactive web application was created to provide key data visualisations and allow users to explore Covid-19 cases across the U.S states. The interactive features allow for a more personalised user experience, to explore as little or as much of the analysed data. In all tabs, the user can use the slider input and drop down list to manoeuvre between dates and states across the United States to visualise how the Covid-19 trajectories changes with these inputs.

Due to the size of this application, I do not have enough memory space to publish this app on ShinyApps.io with my free account. For this reason, the link to my GitHub repo for the Shiny app will be provided.

Motivation of the Shiny App

Tab 1: United States COVID-19 Cases (per 100,000) by State

The dataset for this component originates from the NY Times (Reference).

This component provides the confirmed Covid-19 cases, per 100,000 population that were reported by the state on a given date (Reference to NY Times). The total confirmed cases is plotted on the U.S state map, and is made with a sequential colour scale to establish the geographical location in which the total confirmed cases are the highest. This trend is also plotted on a histogram on the side bar panel, as the pattern can be better articulated since people are generally better at comparing lengths on a common scale as compared to shading or colour saturations ((Cleveland and McGill 1984)).

As a case in point, it can be observed in this tab that after New York surpassed Washington on the 16th March 2020 in the total number of Covid-19 cases, they have since led the United States in this aspect up till this date.

Tab 2: Positive Results in Covid-19 testing

The dataset for this component originates from the NY Times (Reference). Covid-19 testing promotes early detection of community-acquired Covid-19 cases, which can support in public health response to the pandemic such as the allocation of resources for the more vulnerable regions.

This section consists of two components. The first tab measures the percentage of Covid-19 tests returning positive on a given date and is charted on the US state map. Correspondingly, the second segment then depicts the aggregated test results as of the chosen date selected by the user. Not to mention, occasional anomalies (e.g. significant drop in tests or 100% positive test results) can be observed and is generally due to case definitions and non-reporting.

Self Review

What was done well?

The layout of the app was structured well. For instance, I utilised fluidRow(), to manually divide user interface into grids to my liking and tabsetPanel() to combine panels into a stack. Further, I customised the texts in the UI with shing::tags$ functions to build specific HTML tags, it took me a while to figure out the different HTML tags!

Next, reactive programming was done fairly well. To emphasise, due to the long running-tasks embedded in this app, it was not ideal to wait for the app to compute every reactive expression each time the user changes the input. For this reason, I applied observeEvent() function to delay computation of this reactive expression only until the action button is clicked.

Areas of Improvement

Debugging shiny applications is something that I need to work on. Firstly, running reactive codes was problematic for me as I was accustomed to the usual R markdown file system. For this assignment, I created a new Rmd file to make reference to when I ran into errors, which was not the best practice. I should have made use of tools such as reactive graphs, log statements and breakpoints.

Organising large data files to optimise the speed of the Shiny app was another aspect that I needed to improve on. In this case, the joined data set (Covid-19 cases and Map data) consisted of more than 2 million observations. For example, I could have used geom_sf or leaflet() map to plot the map instead of geom_polygon to reduce the object size. By doing it this way, I may not have to join all the map coordinates with the covid-19 data since I could overlay the map with the variables in the Covid-19 data.

Plausible Things I’d like to add

In retrospect, I would add a plot or a new tab that showcase Covid-19 trends over time. In my current Shiny app, Covid-19 patterns were only shown at a point in time, and it may be difficult to compare the trends across countries over time to better understand the data.

I would also like to add some technical details to the app:

  • Plotly Input Events and multiple reactive expressions: Access plotly input events after the map output is generated. For instance, clicking on the state(s) on the U.S. map automatically updates the bar graph to compare the trends of selected states.

  • Cancel long-running tasks while loading: Due to the large amount of data, it takes about 7 seconds or so to load when running locally, and users cannot do anything until the function finish running. I want to add a ‘cancel button’ so users can cancel the computation of reactive expression any time.

  • Pre-populate or download shiny input based on URL parameters As the Covid-19 datasets are updated regularly, I would add a download link for users to directly download data directly from the source (e.g. data from GitHub) to provide a complete and up-to-date data of the ongoing outbreak in the Shiny App.

Limitations of Data/Technology

A limitation to this Shiny App is the credibility of the data. Due to the various state jurisdictions, case definitions and reporting requirements of each U.S state, the reported values may be subjected to significant variations. To illustrate, in the data retrieved from covidtracking.com, numerous US states did not report the negative results of the covid-19 testing.
